# Agent Note Writing SOP

## 目的

后续新增笔记时,必须把用户观点整理成可检查、可复用、可冲突处理的规则。不要只写理念,要写成后续分析可以逐项执行的 SOP。

适用:
- 用户纠正股票 / 缠论 / MACD / 量能 / 做 T / A 股执行规则。
- 用户提供一段复盘、案例、截图或新的交易纪律。
- 需要把一次对话沉淀到 `AGENTS.md`、`CLAUDE.md` 或 `memory/*.md`。

## 文件分层

| 文件 | 作用 | 写法 |
|---|---|---|
| `AGENTS.md` | 最高频运行入口 | 只写强制规则和简版执行口径 |
| `CLAUDE.md` | 完整交易体系 | 写详细背景、流程、表格、索引 |
| `memory/reference_*.md` | 稳定理论 / 长期参考 | 写定义、判定步骤、完整模板 |
| `memory/feedback_*.md` | 单次错误教训 | 写错误、原因、修正、正反例 |
| `MEMORY.md` | 记忆索引 | 每个重要 memory 文件给一句摘要 |

## 标准笔记结构

每条新规则优先整理成这个格式:

```text
# 规则名

## 适用场景
什么时候必须用这条规则。

## 核心定义
这个概念是什么;不能是什么。

## 判定步骤
1. ...
2. ...
3. ...

## 输出要求
必须写哪些字段;禁止哪些模糊说法。

## 正例
这样判断是对的。

## 反例
这样判断是错的;错在哪里。

## 优先级
与旧规则冲突时谁优先。

## 教训案例
股票 / 日期 / 当时误判 / 正确改法。
```

## 缠论笔记必须补齐的字段

缠论相关笔记不能只写“有买点 / 没买点 / 中枢震荡”。必须至少覆盖:

| 字段 | 要求 |
|---|---|
| 操作级别 | 日线 / 60m / 30m / 5m,不能混级别 |
| 分型 / 笔 | 端点、方向、K 数、是否包含处理 |
| 线段 | 由哪些同级别笔构成,是否被破坏 |
| 中枢 | `ZD` / `ZG` / 中轴 / 是否延伸或扩展 |
| 自同构递归 | 本级别结论、次级别支持/反证、能否升级、升级触发位和失败位 |
| 买点标签 | 工具标签 / 人工结构候选 / 原文严格 / 当前执行 |
| MACD | 红绿柱面积、背驰或不背驰、柱体收敛或扩张 |
| 量能 | 放量上涨、缩量回踩、放量下跌、缩量反抽、量价背离 |
| A 股执行 | T+1、仓位、次一交易日止损、可卖股份 |
| 失效位 | 具体价格或结构条件 |

## 缠论输出模板

```text
级别:
最近笔:
线段:
中枢:
自同构递归:
MACD:
量能:
工具标签:
人工结构:
原文严格:
当前执行:
失效位:
下一步验证:
```

图形验证必须附表:

```text
笔表: 起点 / 终点 / 方向 / K数 / 是否确认
中枢表: 构成三段 / ZD / ZG / 当前价位置
背驰表: 前一段MACD面积 / 当前段MACD面积 / 是否背驰
量能表: 离开中枢量能 / 回踩量能 / 结论
```

## 冲突处理

- 用户最新明确纠错优先于旧摘要;必须写日期和改动原因。
- `原文严格买点` 用于定性,`执行试错仓` 用于交易动作;两者不能互相替代。
- 若新规则只是执行层补充,不要删除原文定义;要写清“严格定性不变,执行折扣新增”。
- 如果旧文档出现相反表述,必须同步修正,不能只新增一条让规则自相矛盾。

## 反馈沉淀流程

1. 记录用户原话或核心纠错。
2. 写清当时输出错在哪里。
3. 把纠错翻译成可执行规则。
4. 增加正例和反例。
5. 检查 `AGENTS.md` / `CLAUDE.md` / `MEMORY.md` 是否需要引用。
6. 用 `rg` 搜旧冲突词,修掉明显矛盾。

## 禁止写法

禁止只写:

```text
中枢震荡不要乱买。
缠论要结合 MACD。
候选要看情况。
T+1 注意风险。
```

必须改成:

```text
当价格处于本级别中枢中部:
- 原文严格买点:无
- 人工结构:中枢震荡
- 当前执行:不追、不低吸、不做无优势 T
- 只有靠近 ZD 且小级别背驰,才允许写执行试错仓
- A 股 T+1 下默认 loose 2-3%,次一交易日止损
```

## 核心原则

**理念要落成步骤,判断要落成表格,执行要落成价格和失效位。**
