# A股 实战约束对回测的真实影响

## 教训

回测时假设"理论可执行",但 A股 实战中**每个约束都会扣 PnL**。
sector_jump_v1 加仓策略实测:
- 理论回测: 平均 PnL +6.95%
- A股 真实约束: **+5.27%(缩水 24%)**

## 约束清单 (回测必须扣)

| 约束 | 实际影响 | 单笔扣 |
|---|---|---|
| **T+1** | 当日入场不能当日卖, 止损顺延次日 | 单笔最坏 -32% (跌停叠加) |
| **涨停板 +9.7%** | 入场被拒 + 加仓被拒 | v2 加仓 1460 次被拒/14 月 (约 25% 加仓机会) |
| **跌停板 -9.7%** | 卖出顺延次日开盘 | 101 次卖出顺延/14 月 (单笔亏损扩大) |
| **滑点 0.3%×2** | 买入向上 + 卖出向下 | 0.6% 单次往返 |
| **佣金 0.025%×2** | 双边 | 0.05% |
| **印花税 0.05%** | 仅卖出 | 0.05% |
| **总成本** | 单次往返 | **~0.7%** |

## Why: 实盘单仓必须 ≤ 4% (DIST 状态)

```
理论假设: 单仓 5-8%, 最大单笔亏损 -8% × 5% = 账户 -0.4%

A股 真实最坏:
  跌停日 + T+1 顺延 + 次日继续跌停 = 实际亏损可能 -25%~-40%
  单仓 8% × -40% = 账户 -3.2% (单笔!)
  
解决: 单仓收紧到 4%
  4% × -40% = 账户 -1.6% (可承受)
  
即便 5 次连续止损 = -8% 账户回撤, 仍在欧奈尔派 -10% 月度回撤红线内
```

## How to apply

### 实盘扫描必须叠加 A股 约束:

```python
# 入场时
if 当日 close ≥ 前 close × 1.097:  # 涨停板
    return None  # 拒绝入场, 买不到

actual_entry = close × 1.003  # 含 0.3% 买入滑点

# 止损触发时
if 当日跌停 (low + close 都 ≤ 前 close × 0.903):
    顺延次日开盘卖, 实际亏损可能扩大

# 加仓时
if 当日涨停: 拒加 (买不到)
```

### 仓位规则:

```
A1 主仓:    DIST 4% / 中性 5% / BULL 6-8%
A0 试错:    DIST 2-3% / 中性 3-5% / BULL 5%
单板块:    持仓 ≤ 3 只 (集中度风险)
加仓后:    单仓最大 = 主仓 × 1.5
节假日前:  减仓至总仓位 ≤ 30% (春节/国庆 7 日休市风险)
```

## 验证数据 (sector_jump_v1 + A股 约束 14 月回测)

| 策略 | 理论 PnL | **A股 真实 PnL** | 缩水 |
|---|---|---|---|
| 不加仓 base | +4.14% | **+3.32%** | -20% |
| v1 突破加仓 | +6.74% | **+5.08%** | -25% |
| v2 浮盈加仓 | +6.95% | **+5.27%** | -24% |

最大单笔 (理论 vs 真实):
- 不加仓: -8.0% → -31.8%
- 加仓: -14.7% → -41.3%
