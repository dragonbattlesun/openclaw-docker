# 缠论工程标签与原文买点输出纪律(2026-06-03 整理)

## 教训

用户反馈: “为什么没有 1B 和 2B”。根因不是结构里完全没有候选, 而是输出时把:

- `没有原文严格买点`
- `没有当前位置可执行买点`
- `没有 1B/2B 候选`

混成了一句话: “没有买点”。

这会造成两个反向错误:

1. 把工具或人工观察到的 `1B/2B/3B 候选` 直接说成严格买点, 导致误入场。
2. 因为未达到严格入场标准, 就把已经出现的 `候选低点 / 等待二买 / 等待三买` 省略掉, 导致用户无法跟踪后续确认。

后续必须按 `reference_chanlun_108_core_rules.md` 的层级输出。

## 三层标签

| 层级 | 标签 | 含义 | 能否当入场依据 |
|---|---|---|---|
| 1 | 工具标签 | 扫描器或脚本输出的 `1B/2B/3B` | 不能 |
| 2 | 人工结构候选 | 人工合笔后看到的 `1B候选 / 2B候选 / 3B候选 / 盘整背驰候选 / 小转大候选` | 不能自动入场;只能继续校验,或在 T+1 风险可控时小仓试错 |
| 3 | 原文严格买点 | 满足原文定义的 `严格1B / 严格2B / 严格3B` | 可以按对应级别和风控执行 |

输出时必须同时给出:

```text
工具标签: ...
人工结构: ...
原文严格: 通过 / 不通过, 缺什么
当前执行: 可入场 / 等待 / 不入场 / 只做持仓T
失效位: ...
```

## 统一买点判定

### 左侧买点纪律

更新时间: 2026-06-16。原因: 按 108 课理论固化“缠论允许左侧,但只允许结构完成后的左侧; 不允许下跌途中猜底式左侧”。

适用场景:

- 工具输出 `1B/2B/3B`、人工缠论分析、持仓诊断、下沿回补、盘整背驰、做 T 计划。

核心定义:

- 允许的左侧: 本级别结构已完成后的左侧,即走势类型、笔、中枢、背驰/盘整背驰、反弹笔和失效位都能落地。
- 禁止的左侧: 下跌途中猜底,包括“跌多了”“便宜了”“接近前低”“指标低位”“故事还在”但本级别结构未完成。
- 中枢震荡下沿回补和盘整背驰只能写 `盘整背驰候选 / 中枢下沿回补 / 小级别买点`;不能写成本级别严格 `1B`。

判定步骤:

1. 先定级别,本级别独立判断;小级别只能支持/反证,不能替代本级别买点。
2. 若无本级别 `a+A+b+B+c` 下跌趋势和同级别背驰,不得写严格 `1B`。
3. 若处于本级别中枢震荡,只允许写中枢上下沿短差或更小级别买卖点。
4. 候选试错必须靠近失效位,并写清 A 股 T+1 下新买股份当日不能卖。

输出要求:

```text
工具标签:
人工结构:
原文严格:
当前执行:
失效位:
```

正例:

- `人工结构: 30m 盘整背驰候选 / 中枢下沿回补; 原文严格: 30m 严格1B未成立; 当前执行: 靠近失效位才可 loose 试错。`
- `人工结构: 日线 a+A+b+B+c 下跌趋势背驰候选; 原文严格: 严格1B候选; 当前执行: 左侧小仓或等待2B; 失效位: C 段低点。`

反例:

- `跌到低位,左侧买`。
- `中枢下沿反弹,确认30m一买`。
- `5m 底背驰升级成日线严格1B`。

### 1B

原文严格 `1B` 必须同时满足:

1. 本级别下跌趋势成立, 不是普通中枢震荡。
2. c 段创新低后出现同级别背驰或明确力度衰竭。
3. 反弹笔成立。

允许报告但不能直接交易的标签:

- `1B候选`: 有下跌衰竭、盘整背驰、底分型或反弹笔雏形, 但趋势背驰条件未全。
- `盘整背驰候选`: 仍在中枢震荡里, 只可作为中枢下沿短差、回补候选或小级别买点,不能升级成本级别严格一买。
- `小转大候选`: 小级别底背驰可能引发大级别转折, 但必须等待更大级别三买确认。

禁止:

- 把中枢震荡低点直接叫本级别严格一买。
- 把下跌途中的猜底、超跌、历史低位或单根止跌 K 线叫左侧一买。
- 因为不能入场就说“没有 1B”, 应说“有 1B候选, 但不是原文严格1B”。

### 2B

原文严格 `2B` 必须同时满足:

1. 前面已有有效 `1B`。
2. 之后第一次次级别回调。
3. 回调低点不破 `1B` 低点。
4. 回调不破关键中枢下沿或结构防守位。

允许报告但不能直接交易的标签:

- `2B候选`: 有回踩不破雏形, 但前面 `1B` 尚未严格成立。
- `等待2B`: 已有候选低点, 但还没出现第二次回踩不破。
- `2B失败`: 回踩跌破 1B 低点或关键防守位。

禁止:

- 没有严格 1B 时直接确认严格 2B。
- 把“当前不可以买”写成“没有 2B 路径”。正确写法是“等待回踩不破后才有 2B候选”。

### 3B

原文严格 `3B` 必须同时满足:

1. 次级别走势向上离开本级别中枢 `ZG`。
2. 离开后第一次次级别回试。
3. 回试低点不跌破 `ZG`。

允许报告但不能直接交易的标签:

- `3B候选`: 已突破或正在回试, 但首次回试级别/不破 `ZG` 未确认。
- `等待3B`: 只突破未回踩。
- `3B失败`: 回试跌回中枢或跌破 `ZG`。

禁止:

- 只有突破没有回试就说 3B 成立。
- 后续多次回踩都标成本级别 3B; 第三类买点只看离开后的第一次有效回试。

## 中枢震荡的特殊口径

`reference_chanlun_108_core_rules.md` 明确: 中枢震荡没有本级别一买、二买、三买, 只存在更小级别买卖点。

因此中枢内必须这样写:

```text
本级别: 中枢震荡, 无原文严格 1B/2B/3B。
小级别: 可在中枢下沿看低吸/回补候选, 在上沿看卖点/做T候选。
当前价若在中枢中部: 无当前位置执行买点。
```

不能这样写:

```text
没有买点。
```

应改成:

```text
有/无候选分别说明; 当前无执行买点; 等 xx 位置触发。
```

## 自同构递归输出

缠论走势自同构性要求:不同级别使用同一套结构规则,但买卖点定性不能混级别。

以后所有工程标签解释必须额外写清:

```text
本级别结论: ...
次级别支持/反证: ...
能否递归升级: 可以 / 不可以,缺什么
升级触发位: ...
升级失败位: ...
```

禁止:

- 把 `5m 底背驰` 直接写成 `30m 严格1B`。
- 把 `30m 三买` 直接写成 `日线三买`。
- 用 30m 的笔、5m 的回试、日线的中枢拼一个无级别买点。

正确写法:

```text
30m 本级别仍是中枢震荡,无本级别严格买点。
5m 底背驰只支持 30m 中枢下沿回补候选。
若 30m 后续离开 ZG 并第一次回踩不回中枢,才进入 30m 3B 候选 / 严格校验。
```

## 输出模板

以后个股缠论分析必须包含这个表:

| 级别 | 工具/表象 | 人工结构 | 递归关系 | 原文严格结论 | 当前执行 | 失效位 |
|---|---|---|---|---|---|---|
| 日线 | ... | ... | 次级别是否支持/反证 | ... | ... | ... |
| 60m | ... | ... | 次级别是否支持/反证 | ... | ... | ... |
| 30m | ... | ... | 次级别是否支持/反证 | ... | ... | ... |
| 5m | ... | ... | 能否升级到上级别 | ... | ... | ... |

买点段落必须用这些词:

- `原文严格1B已成立`
- `1B候选, 未严格成立`
- `等待2B`
- `2B候选`
- `等待3B`
- `3B候选`
- `中枢震荡, 无本级别严格买点`
- `当前位置无执行买点`

禁止使用模糊词:

- `有买点`
- `没买点`
- `可以买一点`
- `形态不错`

除非同时给出级别、类型、触发位、失效位。

## 仓位纪律

| 标签 | 仓位口径 |
|---|---|
| 原文严格 1B | 可按低级别试错仓, 必须给候选低点失效位 |
| 原文严格 2B | 可按确认加仓, 跌破 1B 低点失败 |
| 原文严格 3B | 可按趋势仓, 跌回中枢或跌破 `ZG` 失败 |
| 候选 / 等待 / 盘整背驰 | 不给确认仓位; 只给观察、持仓T、触发条件,或 T+1 可承受时的 `执行试错仓 loose 2-3%` |
| 中枢中部 | 不追、不低吸、不做无优势 T |

候选可以报告, 但不能被包装成严格 1B/2B/3B 或确认仓。若用户明确讨论已有仓位做 T, 只能按持仓管理给上下沿和失效位, 不能把它升级成新开仓买点。

## A 股 T+1 执行折扣

A 股不是 T+0。后续所有“试错、做 T、跌破就走”必须加 T+1 折扣:

- 今日买入的新仓或加仓,今日不能卖出;“跌破就走”只能对已有可卖仓位当天执行,对新买股份必须写成次一交易日止损。
- `执行试错仓` 不是原文严格买点。它只表示候选结构已到触发位,且失效位近、隔夜风险可承受,默认仓位 `loose 2-3%`。
- 当前价在中枢中部、上沿追高位、或距离失效位过远时,即使有候选标签,也必须写“等待,不试错”。
- 临近收盘且结构仍有效、量能没有放量破位、MACD 未明显反证时,试错仓可比早盘/盘中更合理;早盘冲高和盘中追高必须更保守。
- 做 T 只能围绕已有可卖股份:先卖已有仓、低位回补会形成新买锁仓;不能把当天回补写成可以同日再卖出的完整 T+0 闭环。
- 候选收益空间不足以覆盖次日低开、跌停、滑点和失效位风险时,不做。

## 示例: 拓维信息复盘口径

错误表达:

```text
没有买点。
```

正确表达:

```text
30m 处于 29.94-30.86 中枢震荡。
29.47 是跌破中枢下沿后的反拉低点, 可标为 30m 盘整背驰/1B候选观察点, 但不是原文严格1B。
当前 30.2 在中枢中部, 无当前位置执行买点。
若后续回踩不破 29.47 并再转强, 才进入 2B候选校验。
若放量离开 30.86 后第一次回踩不回中枢, 才进入 3B候选/严格3B校验。
跌破 29.47, 1B候选失败。
```

## 核心原则

**候选要说清, 严格要校验, 执行要给位, T+1 要折扣。**

不能把候选当严格买点, 也不能把“未达严格买点”简化成“什么都没有”。
